## 功能页面

### 设备

设备页包含设备选择、保持比例的实时画面、控制模式、旋转、鼠标直接触控、硬件按键和设备检查器。检查器提供 Lockdown 设备信息、已安装应用、IPA 安装与卸载，以及描述文件列表。

缺少电脑配对记录的 USB 设备仍会显示在选择器中，并标记为“需要信任”；选择它不会启动必然失败的 CoreDevice 会话。点击“信任设备”，保持 iPhone 或 iPad 解锁，并在设备上确认“信任此电脑”。DeviceHub Mask 会等待这次显式确认，使用安全 Lockdown session 验证生成的记录，通过发现该设备的同一 usbmuxd 或内置 netmuxd 后端保存记录，然后刷新并连接设备。拒绝、设备锁定、断开或 90 秒超时都会明确报告，应用不会自动重复弹出请求。这里的电脑信任记录不同于后续 Wi-Fi CoreDevice 传输使用的 iOS 17+ RemotePairing 凭据。

已配对 USB 会话可通过信息标签页中的“电脑信任”区域显式移除配对，以修复损坏记录。确认后会立即结束控制会话，请求 Lockdown 撤销设备端记录，并通过同一 usbmuxd 后端删除此电脑的私有配对记录。该操作不会抹掉或还原设备，但依赖这些凭据的 Wi-Fi 访问可能停止，重新连接前需要再次在设备上确认“信任此电脑”。结果会区分完整移除、设备未确认但已删除电脑记录，以及设备已撤销但电脑清理失败；最后一种情况应先重新启动设备服务，再重新配对。

检查器的信息标签页会显示 Lockdown 报告的设备与数据区容量，包括用户数据已用和可用空间； 设备名称可直接编辑，后端会建立已配对的 Lockdown session、写入规范化名称，并在确认设备 读回相同值后才报告成功。信息标签页还会通过 Diagnostics Relay 刷新经过隐私过滤的电池诊断，包含电量 与充电状态、循环次数、满充/设计容量、电压/电流、剩余时间和适配器功率。电池及适配器 序列号不会通过私有 API 返回。 信息标签页还会通过 MobileActivationd 读取设备激活状态，必要时使用其 Lockdown 回退。 界面只接收归一化状态，不请求激活记录或激活信息 blob；设备尚未激活时会显示警告，因为 开发者和设备管理服务可能不可用。 信息标签页会优先通过 AMFI 查询开发者模式，并以 MobileImageMounter 作为回退；未启用 时，警告中的操作可通过 AMFI 让“设置 > 隐私与安全性 > 开发者模式”选项显示。该准备 操作不会启用开发者模式、 重启设备或替用户接受设备端确认；这些步骤仍需用户明确完成，因为 DVT 调试与性能服务 依赖最终建立的可信状态。

活动设备为 iPhone 时，“设备信息”标签页还会通过 CompanionProxy 只读查询已配对的 Apple Watch。空列表是有效结果；设备名称、产品类型、watchOS 版本或构建号可能因 iPhone 与 watchOS 版本而缺失。此处不控制 Apple Watch、不启动 Watch 服务，也不转发 Watch 端口。复制或分享配对设备标识符时，应将其视为敏感设备元数据。

画面工具栏始终显示 Home、锁屏、音量加、音量减、静音、Siri 和 Action。标题工具栏提供窗口置顶、设备侧栏与映射侧栏各自独立的显示开关，以及全屏控制。隐藏设备侧栏只会增加画面可用空间，不会拉伸画面，也不会卸载或重置侧栏中正在进行的操作；两个侧栏的显示偏好会分别恢复。切换页面和全屏状态会释放所有输入。在“设备”工作台右键点击手机画面会发送 Home；“按键映射”工作台的右键不会发送 Home。截图按钮会请求原生无损 PNG：优先使用 CoreDevice ScreenCaptureService，失败时回退到 USB Lockdown 或远程 RSD shim 上的 Apple screenshotr 服务。因此截图不依赖视频解码，视频画布尚未恢复时也可以使用；若两个原生服务都暂时不可用，则自动保存当前已解码画布作为后备。画面录制按钮通过系统 WebView 的 MediaRecorder 录制已渲染画布，并按平台支持下载 MP4 或 WebM。离开“设备”页或切换设备时会停止录制。录制只包含画面，不包含通过主机原生输出播放的设备音频。画面工具栏会显示当前已保存的控制配置，无需进入映射编辑器即可切换。设备画面全屏将工具条拆分为高优先级硬件按钮工具条和功能工具条：硬件条完整显示七个硬件按钮，功能条提供重连、输入模式、映射显示、旋转、“更多”和退出；控制配置、显示与采集、音频及系统全屏位于“更多”面板。通过握柄可拖动任一工具条，释放后会吸合到最近的八个稳定边缘槽位之一，并在窗口尺寸变化后保留该槽位。吸合算法会比较工具条实际渲染边界：硬件条始终保留用户指定的位置，发生重叠时功能条自动移动到最近的不重叠槽位；拖动功能条也不会挤走或覆盖硬件条。自动隐藏启用时，指针停留、键盘焦点、拖动或已打开控件会暂停隐藏。设备画面全屏与系统全屏始终是两个互不覆盖的状态。

### 键盘映射

映射页包含放置画布、映射检查器、硬件快捷键和配置管理。编辑器显示源分辨率和按比例 适配后的显示分辨率。背景可选择实时画面或方向正确的冻结截图；截图可以保存为 PNG， 设备断开后仍可用于离线编辑。

可在画布上拖动控件，也可在准确位置右键新增任意 scrcpy-mask 控制器类型。检查器将 控制器与硬件快捷键分栏，支持搜索、复制，以及编辑归一化坐标、组合键、时序、施法中心和 Swipe/MultipleTap 序列点。范围开关可显示全部轨迹和控制范围；编辑时始终显示当前所选 控制器的范围。新建和复制会优先选择当前使用次数最少的 Universal HID 触点身份。编辑后 需要保存配置。即使配置包含更多映射，运行时同时最多选择五个不重复触点身份。

### 应用设置

设置页包含外观、语言、窗口行为、设备音频、剪贴板同步、自动检查更新开关、手动“立即检查”、当前 应用版本和 GitHub 仓库入口。设备音频默认关闭，启用后需重新连接设备生效；播放静音和 音量会通过主机默认原生音频输出立即生效，设备控制工具栏也提供静音按键。播放不依赖 WebView 自动播放权限或页面可见性。剪贴板同步同样默认关闭并在重新连接设备 后生效；启用后，在电脑或设备任一端复制的文本与图片会替换另一端的剪贴板内容。 每次同步会短暂提示方向与内容类型；文本预览会合并空白并限制为 48 个字符。

“设备信息”标签页提供需要二次确认的“重新启动设备”和“关闭设备”，这两项都会结束当前 控制会话；重启后需等待 iOS 与 USB 服务恢复，关机后则必须手动开机。设备画面工具栏的 Lock 会模拟硬件键并可能唤醒已锁定设备；Agent 需要保证单向锁屏时应使用 MCP `lock_device`。

“设备信息”中的“本地设备备份”会通过原生目录选择器，在电脑目录中创建标准、未加密的 MobileBackup2 备份。通常关闭“强制完整备份”，以复用已有备份并只传输变化内容；需要完整 重做时再启用。备份期间请保持设备解锁并持续连接。取消或会话断开后，已经接收的文件会保留， 之后可选择同一目录继续增量备份。备份可能包含 App、账户等私密数据，应按敏感数据保护。 当前版本有意不开放恢复或擦除功能。

同一“设备信息”标签页中的“系统诊断”通过 CoreDevice DiagnosticsService 采集完整 Apple sysdiagnose。选择 `.tar.gz` 目标并确认隐私警告后才会开始；设备端采集可能持续数分钟，之后 才会显示下载进度，典型归档约占用 1–2 GB。期间请保持设备解锁并持续连接。取消、断线或流 错误会删除电脑上的临时文件；只有完整归档刷新并同步后才会替换已有目标。sysdiagnose 可能 包含日志、App 活动、网络元数据及其他设备隐私信息，请按敏感数据保存与共享。

顶层“AFC 文件”工作台提供四个明确范围。“公共 AFC”打开标准媒体容器，通常可包含 `DCIM` 等目录；“App Documents”列出开放文件共享的 App；“App Container”列出可通过 House Arrest 提供完整容器的开发者 App，可使用搜索式 App 选择器指定目标；“崩溃报告”通过 CrashReportCopyMobile 提供搜索、刷新、导出和确认式单报告删除。

公共 AFC 与 App 存储可以导入或导出文件及完整目录树、新建文件夹、重命名，并在二次确认 后递归删除文件夹。导入不会覆盖设备上的同名项目；符号链接和特殊条目不会被继续访问。 文件导出先写入目标同目录的临时文件，目录导出会先完整建立临时目录树，成功后才发布到 所选主机目标。传输期间会锁定范围和 App 选择，以持续保留进度与取消入口。

### 性能

性能工作台将设备侧 DVT 遥测与桌面视频管线指标放在同一界面。设备指标包含系统 CPU、 进程数、Core Animation FPS、GPU 内存、设备聚合网络收发速率以及最近一分钟的观测 连接数。进程能耗表会为当前 CPU 与内存排行中的最多十六个进程分解 Apple Instruments 相对能耗总分、CPU、GPU 和网络分数；这些分数适合同一会话内比较，不代表瓦特。 管线指标包含源帧率、解码与显示帧率、传输带宽、JPEG 编码耗时和帧年龄。 系统 CPU 是将 DVT 报告的聚合 `CPU_TotalLoad` 除以设备逻辑 CPU 核心数后得到的 iPhone 全机百分比，不是桌面主机或单进程 CPU 占用。 “高负载进程”显示 CPU 前十与物理内存前十的并集，可按 CPU 或内存排序。进程 CPU 使用 与系统 CPU 相同的逻辑核心归一化，因此 100% 表示整台设备的总处理能力，而不是单个核心 满载。 “App 活动”按最新优先显示 Apple Instruments 报告的最近二十条应用状态与内存通知。 后端最多保留当前设备会话中 100 条经过净化的事件，建立新会话时清空。事件名称和状态由 设备提供，可能随 iOS 版本变化。

“设备条件模拟”只列出当前 iPhone 实际报告的 DVT 网络和热状态配置。应用条件前必须确认， 因为它会影响整台设备而不只是前台游戏，并可能中断当前控制连接。“恢复正常条件”会停用 当前配置。DeviceHub Mask 在会话结束以及 DVT channel 重连时也会尝试清理。如果界面显示 “尚未确认设备已恢复正常”，请保持设备连接；此时应用还不能证明条件已经清除。可用分组与 配置取决于具体设备和 iOS 版本。

“网络抓包”会显式启动一个有时限的 `pcapd` 会话，并将标准 `.pcap` 文件直接写入原生 保存对话框选择的路径。USB 抓包优先使用 lockdown 服务，Wi-Fi 抓包使用 CoreDevice remote 服务；后者可能被设备或 iOS 策略禁止，出现相应错误时请改用 USB 重连。可以提前停止，最长 五分钟自动结束，并在文件达到 256 MiB 前停止。 设备会话结束或抓包流异常时，已采集的部分仍会收尾为有效 PCAP。数据包正文不会经过 WebView 或 MCP 服务。保存文件可能包含地址、协议元数据和未加密的应用数据，应按敏感 文件处理。

“蓝牙 HCI 抓包”通过设备的 `BTPacketLogger` 服务记录手柄、耳机和其他蓝牙通信，并保存为 Wireshark 可读的 `BLUETOOTH_HCI_H4_WITH_PHDR` PCAP。它同样要求显式启动并在设备会话 结束时自动收尾，最长五分钟、最大 64 MiB。iPhone 必须安装 Apple Bluetooth Logging 配置描述文件；未安装时服务可能生成只有文件头的有效抓包。蓝牙抓包也可能暴露设备活动， 应作为敏感数据处理。原始数据包直接写入选定的主机文件，不会发送到 WebView 或 MCP 服务。

“运行进程”区域会在打开性能工作台时按需执行一次 DVT DeviceInfo 查询，之后只在用户点击“刷新”时再次读取。搜索会匹配经过清理的进程名、App 显示名或 PID。结果只读，最多返回 1,024 个唯一 PID，并标记 iOS 是否将条目归类为 App；它与持续采样的 CPU/内存高占用表相互独立，不提供进程终止、内存读取、可执行路径或 DVT 目录浏览。

只有打开性能工作台时才会启动采样，离开页面后立即停止，避免监控持续增加设备负载。服务健康区域显示设备心跳、虚拟定位、设备条件模拟、设备通知、系统监控、图形监控、网络监控、能耗监控、进程清单、App 活动监控、配对 Apple Watch 发现、主屏幕布局读取、网络抓包、蓝牙 HCI 抓包和系统诊断处于连接、可用、恢复、不可用或停止状态。心跳会在整个活动会话中响应设备的 Lockdown keep-alive；设备休眠、超时或传输失败均由同一有界监督器恢复。单项服务重连不会重建设备视频或输入会话。

“虚拟定位”会自动优先选择 DVT 后端，并在状态旁显示实际选中的后端。DVT 不可用时，会为采用较早开发者磁盘镜像工作流的设备尝试 Apple 传统定位服务；该服务要求已经挂载兼容镜像。定位 channel 恢复时会保留并重放当前坐标；点击“停止模拟”或结束设备会话时会显式恢复真实定位。

## 控制模式

使用设备画面上方的“映射 / 键盘透传”分段控件切换模式。

- **映射**：将物理按键路由到触控映射和硬件按键快捷键。
- **键盘透传**：禁用映射，将 HID 按下和释放事件直接发送到 iOS。支持修饰键、方向键、 导航键、F1-F24 和数字键盘。输入前需要聚焦设备画面。

`Ctrl+Shift+K` 可快速切换模式。切换模式或页面、窗口失焦、进入全屏或设备断开时， 应用会释放全部触点、硬件按键和键盘 usage，避免输入卡住。

键盘透传发送物理 HID 键，不负责文本组合。设备页工具栏的文本工具会把最多 1024 个 Unicode 字符写入设备剪贴板，再发送 Cmd+V；使用前需要先聚焦可编辑文本框。即使持续 剪贴板同步处于关闭状态，这个单次操作也可以使用。

## 直接触控与方向

鼠标输入只在按比例适配后的可见画面内归一化，不包含黑边或画布空白区域。旋转时宽高 使用相同比例，因此横屏和竖屏触控坐标都会与实际画面对齐。

一份 Universal HID report 最多携带五个类型化触点。Rust 后端会在发送到设备前拒绝 重复身份和越界坐标。

## 配置与按键映射兼容性

配置以经过校验的 JSON 文件保存在 Tauri 应用数据目录。支持选择、激活、新建、复制、 重命名、删除、导入和导出；当前激活配置不能删除。

配置可以关联 App Bundle ID。从 DeviceHub Mask 的 App 列表启动已关联 App 时，会自动 激活对应的已保存控制配置。App 列表会显示当前关联，并可将尚未关联的 App 一键关联到 当前配置，或解除当前配置的关联。跨配置关联和冲突需要在配置编辑器中明确处理。

导入弹窗需要明确选择 DeviceHub Mask、scrcpy-mask 或 PlayCover 来源格式，避免存在歧义 的 JSON 文件被错误转换。DeviceHub Mask 可以导入和导出 scrcpy-mask `0.0.1` JSON。全部十三种控制器类型均会 保留，包括嵌套序列坐标、绑定、释放模式、计时和脚本字段。配置兼容不代表支持 Android 传输。

PlayCover 来源支持 `2.0.0` `.playmap` 文件。键盘按钮、键盘拖拽按钮和键盘 方向盘会转换，PlayCover Bundle ID 会成为配置的 App 关联。由于 DeviceHub Mask 尚未提供 等价的输入源，PlayCover 鼠标区域和负数鼠标/手柄键码会被跳过并计入导入提示。运行时仍 限制每份 HID report 最多包含五个同时触点。

硬件快捷键随配置保存。点击快捷键输入框后按键即可绑定，按 Backspace 或 Delete 清除。 触控映射和硬件按键之间不能共用同一个键，两个硬件按键也不能使用同一个键。Siri 等 功能会保留按住时长。

## 应用与描述文件

应用标签页优先通过 CoreDevice AppService 列出应用，必要时回退到 Lockdown Installation Proxy。AppService 进程数据可用时会标记运行中的 App，并支持停止或重启。 此外还支持通过原生文件选择器选择 IPA、显示安装阶段，以及卸载用户确认的可移除应用。 工具栏的 App 范围菜单可以分别加入 Apple 默认 App 与轻 App (App Clips)，并明确标记两种条目；两个范围默认都关闭，隐藏或内部 App 始终不会被请求。设备允许时可以检查或启动系统 App，但界面不会提供卸载入口。轻 App 可以启动和停止，但其生命周期仍由 iOS 管理，界面不会为它提供传统 App 卸载操作。两个可选范围都依赖 CoreDevice AppService；应用会明确报告错误，而不会把 Installation Proxy 更宽泛的目录视为等价回退。 iOS 提供 Installation Proxy 磁盘占用元数据时，每个 App 行会显示安装体积与数据体积之和； 悬停可查看两项明细，排序菜单可将已知总占用从大到小排列，未上报数值的 App 保持在末尾。 这些是设备上报的占用值，并不是对 App 容器进行递归扫描所得。 App 行接近可视区域时才会通过独立的 CoreDevice AppService 连接按需读取真实图标，并在后端把有界 RGBA 数据转换为 PNG；SpringBoardServices 作为兼容回退。两个路径都不可用时继续显示首字母占位，不影响任何管理操作。 同一标签页还会读取经过限制和规范化的 SpringBoard 图标状态，并标记 App 位于 Dock、主屏幕 第几页或哪个文件夹。文件夹和页面位置都是从 1 开始的顺序，不是画面坐标。设备提供图标 度量时，标签页还会显示主网格、文件夹网格、布局尺寸与图标尺寸；这些是 SpringBoard 布局 单位，仍不是准确的 `tap` 坐标。Widget 配置、Web Clip URL 和 SpringBoard 私有标识符会被 省略；任一可选查询失败都不会禁用 App 管理。

点击应用行的文件夹按钮可进入“App 存储”工作台。通过 iOS 文件共享开放内容的 App 默认 进入 Documents，开发签名 App 可切换到完整容器；分段控件可在 iOS 允许时切换 House Arrest vend 出的 Documents 根目录与完整 App 数据容器。这里可以浏览目录、上传或下载本地 文件和完整文件夹、新建文件夹、重命名，以及在确认后删除文件或整个目录树。递归操作拒绝 符号链接和特殊条目，并限制为 64 层、100,000 个条目；上传不会覆盖同名项目。 修改完整容器可能破坏 App 数据，因此只应对可恢复的调试目标使用此范围。既未开放文件共享 也没有开发容器的 App 不显示存储按钮。 上传或下载期间，工作台会显示已传输字节和项目数量；单文件还会显示精确百分比。文件夹 使用持续更新的计数，因为传输前不会为了计算总量重复扫描整个目录。 传输栏中的停止按钮可以取消当前上传或下载。主机或 App 容器中的临时数据清理完成前， 工作台会保持忙碌状态。

“App 存储”与“设备公共文件”是两套独立边界：前者通过 House Arrest 进入所选 App vend 出的 Documents 或完整容器；独立的 **AFC** 标签页使用设备级标准 AFC 公共媒体容器。 两者实际可见内容及写权限仍由 iOS 控制。AFC 路径栏可直接输入绝对路径或相对根目录的 路径；排序菜单始终保持目录优先，并可按名称、大小或修改时间排列各类条目。切换标签页会 保留当前路径和排序，切换设备则将工作台重置到根目录。 导入和导出会显示实时传输字节与项目数量；单文件还会显示精确百分比。目录操作使用持续 更新的计数，避免重复扫描设备目录树。 传输栏中的停止按钮可以取消当前导入或导出。主机或设备端临时数据清理完成前，工作台会 保持忙碌状态。

卸载前后端会再次查询设备元数据，不会把前端状态作为授权依据。切换设备或结束会话会 取消正在进行的安装或卸载。IPA 上传阶段暂时显示不确定进度，直到 Installation Proxy 开始报告设备端安装百分比。

描述文件通过 Misagent 读取并作为 CMS SignedData 解码。界面只接收标准化元数据，不会 收到原始文件或设备标识符。损坏的描述文件会作为单独错误行显示。安装操作只接受绝对 路径下、非空且不超过 16 MiB 的 `.mobileprovision` 文件；后端会先校验 CMS 载荷、plist 元数据、UUID 和有效期，再向设备发送内容。安装后还会重新查询设备目录，确认对应 UUID 确实存在。

移除描述文件必须二次确认，因为仅由该描述文件签名的 App 可能无法继续启动。移除前， 后端会刷新设备目录，只接受格式合法且当前确实已安装的 UUID；移除后再次刷新，若文件 仍然存在则明确报告冲突。

“崩溃报告”标签页会刷新并递归列出 CrashReportCopyMobile 暴露的报告，支持按报告名称和设备路径搜索。导出使用原生保存对话框，把选中的普通文件写入主机目标路径。只有点击单项危险操作并在对话框中确认后才会永久删除报告；后端会在删除前重新校验绝对路径与普通文件元数据。报告正文不会发送到 WebView；桌面导出会拒绝路径穿越、非文件条目和超过 128 MiB 的报告。MCP 保持只读，可显式读取通过校验的报告路径，但最多返回 1 MiB，供 agent 诊断。

## 设备日志

"设备日志"工作台优先显示当前 iPhone 的结构化统一日志；服务不可用时自动回退到 SyslogRelay。设备提供信息时，统一日志会包含级别、进程、PID、子系统、类别和源文件； 搜索同时覆盖正文与这些上下文，并可按通知、信息、调试、错误或故障级别过滤。工作台还 支持暂停显示、自动滚动、复制当前可见结果以及清空有界内存缓冲区。日志只在该工作台 打开时采集，离开页面后会停止；当日志量过大导致旧条目离开缓冲区时，界面会显示警告。

“导出日志归档”使用独立的 OsTraceRelay 连接创建离线原始 tar 归档，不会中断实时日志。选择最近 1、6 或 24 小时，确认隐私提示后再选择 `.tar` 目标。导出向 iOS 请求最多 256 MiB，应用另行强制 512 MiB 硬上限和十分钟截止时间，并允许取消。只有非空、块对齐且具备预期 tar 结束块的归档完成刷新与同步后才会替换目标；取消、断线、超时、格式异常或失败都会删除临时文件。归档可能包含 App 活动、账户标识符、网络元数据及其他敏感设备信息，其内容不会进入 WebView 私有 API 或 MCP 传输。

## 语言与字体

界面支持简体中文（`zh-CN`）和英语（`en-US`）。首次运行时，`zh-*` 系统区域使用中文， 其他区域使用英语。语言切换立即生效，并保存在 `devicehub-mask.locale`。

React 界面和 Ant Design 都使用系统字体。切换语言不会重写已有配置名称或用户标签。

## 更新

可在设置页关闭自动 nightly 检查，手动检查按钮始终可用。确认更新后，应用会下载、验证 签名、安装并重启。签名和发布机制请查看[发布与更新](distribution.md)。

## MCP 自动化

DeviceHub Mask 运行期间，MCP 客户端可以连接 Streamable HTTP 端点 `http://127.0.0.1:8009/mcp`。服务提供截图、点击、滑动、文本与按键输入、硬件按键、 同步多点触控、App 查询、启动/重启与停止、旋转、设备选择与重连、虚拟定位、画面帧同步、 设备条件、单向锁定设备、崩溃诊断和会话状态等工具。`lock_device` 使用 Diagnostics Relay 的休眠请求，不会像硬件锁定按键切换那样唤醒已经锁定的设备。

`device_details` 会刷新当前设备的产品与系统版本、硬件型号、存储、激活状态、开发者模式 和电池诊断。默认省略 UDID、序列号和 ECID；只有确实需要稳定硬件身份时才应启用 `include_identifiers`。`list_apps` 默认返回用户安装的 App；设置 `include_system=true` 可请求 Apple 默认 App，设置 `include_app_clips=true` 可请求轻 App，也可以同时启用得到合并目录。这些可选范围依赖 CoreDevice AppService，且不会暴露隐藏或内部 App。每个结果都会保留 `is_app_clip`，使 agent 能区分由 iOS 管理的轻 App 与传统已安装 App。 `wait_for_device_event` 无需轮询即可等待 App 安装/移除、激活状态、 磁盘用量、设备名称或锁屏状态变化等归一化事件。下一次调用应把返回的 `sequence` 作为 `after_sequence`，避免订阅竞态；工具不会返回原始通知名称或载荷。

设备激活状态通知也会刷新“设备信息”标签页。收到 SpringBoard 锁屏状态通知时，应用会 释放所有活动的桌面输入，并将事件公开为 `lock_state_changed`。Notification Proxy 不携带 变化后的锁定值，因此 agent 必须重新截图观察当前画面，不能把该事件本身解释为已锁定或 已解锁。

`list_companion_devices` 使用与“设备信息”相同的有界、只读 CompanionProxy 查询，返回 活动 iPhone 已配对 Apple Watch 的可用元数据。空结果是有效状态；该工具不提供 Watch 控制、服务启动或端口转发。配对设备标识符应按敏感设备元数据处理。

`home_screen_layout` 返回与 App 标签页相同的 Dock、页面和文件夹路径。这些位置只用于判断 App 所在区域，不能作为 `tap` 的像素坐标；agent 仍需通过 `screenshot` 进行视觉定位。该工具 在可用时通过 `metrics` 增加设备报告的屏幕布局与图标尺寸、主屏/文件夹行列数、Dock 容量和 页面上限；它们描述 SpringBoard 布局单位，不是当前截图坐标空间。工具不会返回原始 icon-state plist、Widget 配置、Web Clip URL 或 SpringBoard 私有 UUID。

`list_device_conditions` 返回与性能工作台相同的有界 DVT 网络/热状态目录、当前活动配置和 清理状态。`apply_device_condition` 只接受该目录中存在的组与配置组合，并会影响整台设备， 因此既可能中断前台游戏，也可能中断 MCP 连接。测试结束后必须调用 `clear_device_condition`。若传输故障后仍显示清理待确认，请保持设备连接，让监督器在 DVT channel 恢复后还原正常条件。

`performance_snapshot` 会在调用期间临时启用现有 DVT 性能服务，返回 CPU、高负载进程、 能耗、图形、GPU 内存和网络指标。默认最多等待 2.5 秒获取新样本；将 `wait_ms` 设为 `0` 可立即读取缓存快照。`recent_device_logs` 会临时启用设备日志服务，每次最多返回 500 条； 使用 `after` 序列游标可增量读取，也可按 `level` 或不区分大小写的 `query` 筛选。MCP 的 临时需求不会关闭桌面界面已经启用的性能采样或日志流。

App 意外退出后，可调用 `list_crash_reports` 获取按时间倒序的元数据，并按报告名称或设备 路径筛选。将其返回的准确 `device_path` 传给 `read_crash_report`；默认返回 256 KiB， 硬上限为 1 MiB，并携带 `truncated` 和 `lossy_utf8` 标记。该工具只读，并拒绝相对路径、 路径穿越、目录和超限请求。

当 WebDriverAgent 已在活动设备上完成安装、签名并运行时，`wda_status` 可探测设备端口 8100。如果已安装开发者 App 的 Bundle ID 以 `.xctrunner` 结尾，可通过 App 列表行上的 WDA 按钮或 MCP `wda_start` 显式使用 XCTest 启动，启动等待上限为 30 秒。 设备必须已启用开发者模式并挂载匹配的开发者磁盘镜像，“设备信息”会显示当前镜像状态。 未挂载时可点击“挂载镜像”并按原生文件选择器依次选择本地文件。iOS 16 及以前需要 `DeveloperDiskImage.dmg` 和对应 `.signature`；iOS 17 及以后需要来自同一兼容镜像集的 DMG、`.trustcache` 与 `BuildManifest.plist`。只能使用与当前 iOS build 匹配的可信文件； 个性化挂载会在确认后连接 Apple 签名服务。应用不会自动查找或下载镜像，取消上传后可能 需要从头重试完整挂载。 `wda_runner_status` 只报告由 DeviceHub Mask 监督的 Runner，`wda_stop` 也只停止该 Runner， 不会终止外部启动的 WDA。设备会话结束时会自动清理 Runner。DeviceHub Mask 不负责安装、 签名或静默自动启动 WDA。

镜像已挂载时，可通过“设备信息”中的同一控制区显式卸载。卸载可能立即终止正在运行的 XCTest 与 WebDriverAgent 会话，因此应先停止由应用管理的自动化。挂载、卸载和取消请求会 串行执行；切换设备会取消当前镜像操作。

`wda_ui_tree` 返回长度受限的辅助功能 XML 树，`wda_find_elements` 返回带画面矩形的 零基匹配结果，`wda_click` 则在一次请求中查找并点击指定结果。支持的策略为 `accessibility id`、`name`、`class name`、`xpath`、`-ios predicate string` 和 `-ios class chain`。应优先使用辅助功能 ID 或名称；复杂页面上的宽泛 XPath 查询可能较慢。

首次语义请求按需建立 WDA 会话； 设备断开或请求失败时会丢弃该会话，而不会重启主画面控制会话。UI 树可能包含密码、消息及 其他可见文本，应按敏感 MCP 输出处理。未提供有效辅助功能元数据的 App 仍需使用截图坐标 控制。

`type_text` 用于可打印 ASCII HID 逐键输入；CJK 或其他 Unicode 文本应使用 `paste_text`，它会等待设备剪贴板写入和 Cmd+V 完成后再返回成功。

发送坐标输入前应先截图，并把返回的 `image_width` 和 `image_height` 传给 `tap` 或 `swipe`、`multi_touch`，这样截图缩放后坐标仍然准确。对延迟敏感的游戏操作可关闭动作 的 `wait_for_settle`，再把返回的 `frame_version_after` 传给 `wait_for_frame`，等待新画面 后继续截图。MCP 复用桌面应用当前的设备会话，不会再建立一条与手机竞争的连接。

例如，`multi_touch` 可以在同一段 250ms HID 手势中推动左侧摇杆并按住右侧动作按钮：

```json
{
  "contacts": [
    { "x1": 180, "y1": 700, "x2": 240, "y2": 650 },
    { "x1": 850, "y1": 680, "x2": 850, "y2": 680 }
  ],
  "duration_ms": 250,
  "image_width": 1024,
  "image_height": 768
}
```

该端点没有鉴权，设备截图、性能进程名称、设备日志和崩溃报告都可能包含敏感信息。除非 主机位于可信隔离网络，否则应保持监听回环地址。开发者可用 `DEVICEHUB_MCP_ADDR` 修改监听地址，详见[开发指南](development.md)。
